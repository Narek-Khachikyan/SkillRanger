# Browser Evidence

Capture the fixed 390, 768, and 1440 pixel viewport matrix for every declared state. Each evidence bundle records route, state, viewport, source identity, adapter capabilities, screenshot path, browser/runtime errors, keyboard and focus checks, accessibility findings, overflow and overlap checks, and mechanical measurements. Screenshots must be non-empty, local artifacts. Host adapters report their command, capabilities, capture timestamp, and failures; unavailable browser or screenshot capabilities produce an unverified result, never a verified claim.
